# 외부 접속 빠른 시작 가이드

## 5분 안에 외부 접속 설정하기

### 방법 1: 간단한 포트 포워딩 (개발/테스트용)

#### Windows

1. **방화벽 규칙 추가**
   ```powershell
   # 관리자 권한으로 실행
   New-NetFirewallRule -DisplayName "Open WebUI" -Direction Inbound -LocalPort 8080 -Protocol TCP -Action Allow
   ```

2. **서버 IP 확인**
   ```powershell
   ipconfig
   # IPv4 주소 확인 (예: 192.168.1.100)
   ```

3. **라우터 설정**
   - 브라우저에서 `192.168.1.1` 접속 (라우터에 따라 다름)
   - 포트 포워딩 메뉴 찾기
   - 외부 포트: 8080, 내부 IP: 서버 IP, 내부 포트: 8080

4. **공인 IP 확인**
   ```powershell
   Invoke-RestMethod -Uri "https://api.ipify.org?format=json"
   ```

5. **접속**
   - 같은 네트워크: `http://192.168.1.100:8080`
   - 외부 네트워크: `http://공인IP:8080`

**⚠️ 주의**: 이 방법은 보안이 약합니다. 프로덕션 환경에서는 HTTPS와 인증을 반드시 사용하세요.

### 방법 2: Nginx 역방향 프록시 (권장)

#### Linux (Ubuntu/Debian)

```bash
# 1. Nginx 설치
sudo apt update
sudo apt install nginx

# 2. 설정 파일 생성
sudo nano /etc/nginx/sites-available/open-webui

# 다음 내용 입력:
server {
    listen 80;
    server_name your-domain.com;  # 또는 IP 주소

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# 3. 활성화
sudo ln -s /etc/nginx/sites-available/open-webui /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx

# 4. 방화벽 설정
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

#### Windows

1. Nginx 다운로드: https://nginx.org/en/download.html
2. `nginx.conf` 수정 (위 설정 참고)
3. `nginx.exe` 실행

### 방법 3: 클라우드 배포 (가장 안전)

#### AWS EC2 (예시)

1. **EC2 인스턴스 생성**
   - Ubuntu Server 선택
   - 보안 그룹: HTTP(80), HTTPS(443) 열기

2. **서버 설정**
   ```bash
   ssh -i your-key.pem ubuntu@your-ec2-ip
   sudo apt update
   sudo apt install python3 python3-pip nginx
   ```

3. **Open WebUI 설치 및 실행**
   ```bash
   # (로컬 설치와 동일)
   ```

4. **Nginx 설정** (위 방법 2 참고)

5. **접속**: `http://your-ec2-public-ip`

## 문제 해결

### 접속이 안 될 때

1. **서버 실행 확인**
   ```bash
   curl http://localhost:8080
   ```

2. **포트 확인**
   ```bash
   netstat -tuln | grep 8080
   ```

3. **방화벽 확인**
   ```bash
   sudo ufw status
   ```

4. **로그 확인**
   ```bash
   # 서버 로그
   tail -f /var/log/open-webui.log
   
   # Nginx 로그
   tail -f /var/log/nginx/error.log
   ```
